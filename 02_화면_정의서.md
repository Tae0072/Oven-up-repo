# 02. 화면 정의서

> 오븐업 화면 목록과 이동 흐름. **손님용**과 **사장님용(관리자)** 으로 나뉜다.
> 버전: v0.2 · 작성일: 2026-07-02 (개정: 홈/메뉴 분리, 소셜로그인, 예약·단체주문·고객의소리, 관리자 추가)

---

## 1. 화면 흐름 한눈에 보기

### 손님용
```
[로딩(스플래시)]  ← 로고 애니메이션. 로그인 상태면 홈으로 바로 이동
        │
        ▼
[로그인/회원가입]  ← 자체 + 카카오 + 네이버
        │
        ▼
     [홈 (홈페이지)]  ← 브랜드 소개 · 배너 · 각 메뉴로 가는 바로가기 · 하단 내비(홈·메뉴·마이페이지)
        │
        ├──▶ [메뉴 목록] ──▶ [메뉴 상세·옵션] ──▶ [장바구니] ──▶ [주문서] ──▶ [결제] ──▶ [주문 완료]
        │
        ├──▶ [예약 주문]
        ├──▶ [단체 주문]
        ├──▶ [고객의 소리]
        │
        └──▶ [마이페이지] ──▶ 주문 내역 · 쿠폰 · 적립 · 예약 내역
```

### 사장님용 (관리자)
```
[관리자 로그인] ──▶ [관리자 홈(대시보드)]
        ├──▶ [메뉴 관리]        (등록/수정/품절)
        ├──▶ [주문 관리]        (접수/상태 변경)
        ├──▶ [예약 관리]
        ├──▶ [단체주문 관리]
        ├──▶ [고객의 소리 관리]  (답변)
        └──▶ [쿠폰 관리]        (발급/설정)
```

## 2. 손님용 화면 목록

| 번호 | 화면 이름 | 하는 일 |
| --- | --- | --- |
| S0 | **로딩(스플래시)** | 앱 시작 시 로고 애니메이션. 로그인 상태면 홈, 아니면 로그인으로 이동 |
| S1 | 로그인 / 회원가입 | 이메일 가입/로그인 + 카카오·네이버 로그인 |
| S2 | **홈 (홈페이지)** | 브랜드 소개, 배너, 각 기능으로 가는 시작 화면 |
| S3 | **메뉴 목록** | 카테고리별 메뉴 페이지 |
| S4 | 메뉴 상세 | 사진·설명·가격·옵션 선택 후 장바구니 담기 |
| S5 | 장바구니 | 담은 메뉴 확인, 수량 변경/삭제, 총액 |
| S6 | 주문서 작성 | 매장/포장, 요청사항, 쿠폰·적립 사용 |
| S7 | 결제 | 결제 수단 선택 후 실제 결제 |
| S8 | 주문 완료 | 주문 성공 안내, 주문번호 |
| S9 | 예약 주문 | 날짜·시간 지정 → 선결제 → 픽업/배달 |
| S10 | 단체 주문 | 다수 인원 주문 요청/견적 문의 |
| S11 | 고객의 소리 | 문의·건의 작성 및 답변 확인 |
| S12 | 마이페이지 | 주문 내역, 쿠폰함, 적립 현황, 예약 내역 |

## 3. 손님 화면별 상세

### S0. 로딩(스플래시) 🆕
- **성격**: 앱을 켜면 **가장 먼저** 잠깐 보이는 로고 화면. 브랜드 첫인상 + 로그인 상태 확인 시간.
- **요소**: 브랜드색 배경 + 5VEN UP 로고(페이드 인 + 살짝 확대 애니메이션).
- **동작**: 애니메이션을 잠깐 보여준 뒤 자동 이동 — **로그인 유지 상태면 홈(S2)**, 아니면 **로그인(S1)** 으로.

### S1. 로그인 / 회원가입
- **요소**: 이메일·비밀번호 입력, [로그인], [회원가입], **[카카오로 시작하기]**, **[네이버로 시작하기]** 버튼
- **동작**: 로그인 성공 → 홈(S2). 소셜 로그인은 카카오/네이버 창으로 이동 후 자동 로그인.
- **참고**: 소셜 로그인은 카카오/네이버 개발자 사이트에 앱 등록이 필요(뒤에서 안내).

### S2. 홈 (홈페이지) 🆕
- **성격**: 앱을 열면 처음 보이는 **대문(랜딩) 화면**. 메뉴 목록이 아니라 브랜드 첫인상.
- **요소**: 대표 배너/이미지, 매장 소개 한 줄, **바로가기 버튼**(메뉴 주문 / 예약 / 단체 주문 / 고객의 소리), 하단 내비게이션(홈·메뉴·마이페이지)
- **동작**: 각 바로가기 → 해당 페이지로 이동

### S3. 메뉴 목록 🆕(홈에서 분리)
- **성격**: **메뉴만 보여주는 별도 페이지**. 홈의 "메뉴 주문"에서 진입.
- **요소**: 상단 카테고리 탭(커피/티/디저트), 메뉴 카드(사진·이름·가격), 장바구니 아이콘(담긴 개수)
- **동작**: 메뉴 카드 클릭 → 메뉴 상세(S4)

### S4. 메뉴 상세
- **요소**: 큰 사진, 이름, 설명, 기본가, 옵션 선택(사이즈/샷 추가/얼음), 수량, [장바구니 담기]
- **금액**: (기본가 + 옵션가) × 수량

### S5. 장바구니
- **요소**: 담은 항목 리스트, 수량 +/- , 삭제, 총액, [주문하기]

### S6. 주문서 작성
- **요소**:
  - 주문 요약
  - **수령 방식**: 매장식사 / 포장(픽업) / **배달**
  - **수령 시점**: 지금 주문 / **예약 주문(날짜·시간 지정)**
  - 배달 선택 시 **배달 주소** 입력 (명지에코펠리스 건물만 가능)
  - 요청사항, **쿠폰 선택**, **적립금 사용**, 최종 금액
  - [결제하기]
- **배달 조건 안내**: 배달은 **명지에코펠리스 건물 내 + 샌드위치 2개 이상**일 때만 선택 가능. 조건 미충족 시 픽업으로 안내(기능서 §6 참고).
- **동작**: [결제하기] → 결제(S7). 예약 주문도 지금 결제(선결제).

### S7. 결제
- **요소**: **결제 수단 선택** — 카드 / 카카오페이 / 네이버페이 / 토스페이 / 삼성페이, 최종 금액, [결제]
- **동작**: 결제 대행사(PortOne 등) 결제창 → 성공 시 주문 완료(S8) + **적립 지급**
- **참고**: 여러 간편결제를 PortOne 한 곳으로 연동해 결제창에서 함께 노출.

### S8. 주문 완료
- **요소**: 접수 안내, 주문번호, [주문 내역 보기]
- **참고**: 이후 상태가 바뀌면 **푸시 알림**으로 알려줌(예: 준비완료).

### S9. 예약 주문 🆕
- **성격**: 좌석 예약이 아니라 **날짜·시간을 정해 미리 주문·결제하고, 그 시간에 픽업/배달**받는 예약 주문.
- **흐름**: 메뉴 담기 → 주문서(S6)에서 **예약 주문 + 수령 방식(픽업/배달)** 선택 → 날짜·시간(배달이면 주소) 지정 → **결제(S7)**
- **요소**: 날짜·시간 선택기, 수령 방식, (배달 시)주소, 주문 항목, 결제
- **동작**: 선결제 완료 → 마이페이지 주문 내역에 "예약 주문"으로 표시. 지정 시간에 준비/배달.
- **상태**: 결제완료 → 준비중 → (픽업대기 / 배달중) → 완료

### S10. 단체 주문 🆕
- **성격**: 인원이 많은 주문(회사·모임 등)을 미리 요청/문의.
- **요소**: 희망 날짜·시간, 인원/수량, 메뉴 대략 선택 또는 요청사항, 연락처, [문의 보내기]
- **동작**: 사장님이 확인 후 견적/확정 연락. (즉시 결제 아님, 협의형)

### S11. 고객의 소리 🆕
- **요소**: 제목, 내용, (선택)사진 첨부, [보내기]. 내가 보낸 글 목록과 **사장님 답변** 확인.
- **동작**: 접수 → 사장님 답변 시 알림.

### S12. 마이페이지 🆕
- **요소**: 내 정보, **주문 내역**(상태별), **쿠폰함**, **적립 현황**, **예약 내역**, 로그아웃
- **동작**: 각 항목 클릭 시 상세

## 4. 사장님용(관리자) 화면 목록

| 번호 | 화면 이름 | 하는 일 |
| --- | --- | --- |
| A1 | 관리자 로그인 | 사장님/직원 전용 로그인 |
| A2 | 대시보드 | 오늘 주문·매출·신규 예약/문의 요약 |
| A3 | 메뉴 관리 | 메뉴 등록·수정·삭제, 품절 처리 |
| A4 | 주문 관리 | 들어온 주문 접수, 상태 변경(준비중→준비완료 등) |
| A5 | 예약주문 관리 | 시간대별 예약 주문 확인, 준비/픽업/배달 처리 |
| A6 | 단체주문 관리 | 단체 문의 확인·답변·확정 |
| A7 | 고객의 소리 관리 | 문의에 답변 등록 |
| A8 | 쿠폰 관리 | 쿠폰 생성·발급·설정 |

### 관리자 화면 상세 요점
- **A3 메뉴 관리**: 메뉴 사진 업로드, 이름·가격·설명·카테고리 입력, 옵션 추가, 품절 토글.
- **A4 주문 관리**: 새 주문 실시간 확인, 상태 버튼(접수→준비중→준비완료→픽업완료). 상태 변경 시 손님에게 **푸시 알림** 발송.
- **A5 예약주문 관리**: 예약 시간대별로 주문 확인, 준비 시작·픽업 안내·배달 출발 처리, 손님에게 알림.
- **A7 고객의 소리 관리**: 문의별 답변 작성 → 손님 알림.

> 관리자 화면은 **손님용 핵심 흐름이 돌아간 뒤** 붙이는 걸 권장하지만, 이번 프로젝트 범위에 포함한다(로드맵 참고).

## 5. 우선순위 (개발 순서 제안)

입문자 기준, 한 번에 하나씩:

**1차 (손님 핵심 흐름)**
1. S2 홈 → S3 메뉴 목록 (가짜 데이터)
2. S4 메뉴 상세 → S5 장바구니
3. S6 주문서 → S8 주문 완료
4. S1 로그인 (자체 로그인 먼저) → 이후 카카오·네이버 추가
5. S12 마이페이지 · 주문 내역

**2차 (부가 손님 기능)**
6. S9 예약, S10 단체 주문, S11 고객의 소리

**3차 (관리자 + 결제 + 알림 + 혜택)**
7. A1~A4 관리자 (메뉴·주문 관리)
8. S7 결제 연동
9. 푸시 알림
10. 쿠폰·적립 (S6 사용 + A8 관리)
11. A5~A8 관리자 나머지

> 소셜 로그인/결제/푸시는 외부 서비스 등록·심사가 필요해 시간이 걸릴 수 있어, 화면·흐름을 먼저 만든 뒤 붙이는 게 안전하다.
