# 03. 기능 명세서

> 각 기능이 "정확히 어떻게 동작하는지" 정리. 개발할 때 이 문서를 기준으로 만든다.
> 버전: v0.2 · 작성일: 2026-07-02 (개정: 소셜로그인·예약·단체주문·고객의소리·관리자·푸시·쿠폰/적립 추가)

---

## 1. 회원 (로그인/회원가입/소셜로그인)

| 기능 | 설명 |
| --- | --- |
| 회원가입 | 이메일, 비밀번호, 이름, 전화번호로 계정 생성 |
| 로그인 | 이메일·비밀번호 확인 후 로그인(토큰 발급) |
| **카카오 로그인** | 카카오 계정으로 로그인. 첫 로그인 시 자동 회원 생성 |
| **네이버 로그인** | 네이버 계정으로 로그인. 첫 로그인 시 자동 회원 생성 |
| 로그아웃 | 로그인 상태 해제 |

**규칙**
- 이메일 중복 불가, 비밀번호 최소 8자, 비밀번호는 **암호화(해시)** 저장.
- 소셜 로그인은 **OAuth** 방식: 카카오/네이버가 대신 인증해주고, 우리는 결과만 받는다.
- 같은 이메일이면 자체 계정과 소셜 계정을 **하나의 회원으로 연결**할 수 있게 설계(선택).

**소셜 로그인 준비물 (개발 시)**
- 카카오: [카카오 developers](https://developers.kakao.com) 앱 등록 → REST API 키, 리다이렉트 URI 설정
- 네이버: [네이버 developers](https://developers.naver.com) 앱 등록 → Client ID/Secret, 콜백 URL 설정

## 2. 홈 & 메뉴 조회

| 기능 | 설명 |
| --- | --- |
| 홈(홈페이지) | 브랜드 소개·배너·바로가기를 보여주는 시작 화면 (메뉴 목록과 분리) |
| 메뉴 목록 | 카테고리별 판매중 메뉴를 **별도 페이지**로 보여준다 |
| 메뉴 상세 | 특정 메뉴의 사진·설명·가격·옵션 |

**규칙**
- "판매중지/품절" 메뉴는 목록에서 안 보이거나 "품절" 표시.
- 옵션 예시: 사이즈(Tall/Grande), 샷 추가(+500), 얼음 양.

## 3. 장바구니

| 기능 | 설명 |
| --- | --- |
| 담기 | 메뉴 + 옵션 + 수량 추가 |
| 수량 변경 / 삭제 | 항목 수량 조절·제거 |
| 금액 계산 | (기본가 + 옵션가) × 수량 합산 |

**규칙**
- 옵션이 다르면 다른 항목. 수량 0이면 자동 삭제.
- **장바구니는 서버에 저장**한다(회원별). 앱을 껐다 켜거나 다른 기기(웹↔태블릿)에서도 담아둔 내용이 유지된다.
- 주문 시 서버 장바구니 내용으로 주문을 만든다.

## 4. 주문

| 기능 | 설명 |
| --- | --- |
| 주문 생성 | 장바구니 내용으로 주문 생성, 주문번호 발급 |
| 주문 옵션 | 수령 방식(매장/포장/배달), 수령 시점(지금/예약), 요청사항 (§6 참고) |
| 주문 상태 | 결제완료 → 준비중 → 준비완료 → (픽업완료 / 배달중 → 배달완료) |

**규칙**
- 주문 시점의 **가격을 그대로 저장**(나중에 메뉴 가격이 바뀌어도 과거 주문은 안 변함).
- 결제 완료돼야 정식 주문. 결제 전에는 "결제대기".
- 상태가 바뀌면 손님에게 **푸시 알림** 발송(§8 참고).

## 5. 결제

| 기능 | 설명 |
| --- | --- |
| 결제 요청 | 주문 금액만큼 결제 대행사(PortOne 등) 결제창 호출 |
| 결제 검증 | 결제 후 서버가 대행사에 실제 결제 여부·금액 재확인 |
| 결제 실패 | "결제실패"로 두고 재시도 가능 |

### 지원 결제 수단 🆕
손님이 결제창에서 아래 중 하나를 고른다.

| 결제 수단 | 비고 |
| --- | --- |
| 신용/체크카드 | 기본 |
| **카카오페이** | 간편결제 |
| **네이버페이** | 간편결제 |
| **토스페이** | 간편결제 |
| **삼성페이** | 간편결제 |

> 💡 **입문자 핵심 팁**: 이 결제수단들을 각각 따로 연동하지 않는다. **PortOne(구 아임포트) 한 곳에 연동하면** 카카오·네이버·토스·삼성페이·카드를 결제창에서 한 번에 제공할 수 있다. (각 페이사와 개별 계약이 필요한 경우도 있으니, 시작은 카드+카카오페이 정도로 열고 점차 늘리는 걸 권장.)

**규칙 (중요)**
- 결제 금액은 **서버가 계산한 금액** 기준(화면 값 신뢰 금지, 조작 방지).
- 카드번호 등 민감정보는 우리가 저장하지 않음(대행사 처리).
- 결제 성공 시 **적립 지급**, 쿠폰 사용분 차감(§10 참고).
- **예약 주문·배달 주문도 동일한 결제 흐름**을 쓴다(§6 참고).

## 6. 주문 수령 방식 & 예약 주문 🆕

주문은 **수령 방식**과 **수령 시점**을 고른다. 예약은 좌석 예약이 아니라 **"미리 시간을 정해 주문·결제하고, 그 시간에 픽업/배달받는 예약 주문"** 이다.

### 수령 방식 (fulfillment)
| 방식 | 설명 | 추가 정보 |
| --- | --- | --- |
| 매장식사 | 매장에서 먹기 | - |
| 포장(픽업) | 손님이 매장에서 가져감 | - |
| **배달** 🆕 | 입력한 주소로 배달 | 배달 주소, (선택)배달비 |

### 수령 시점
| 시점 | 설명 |
| --- | --- |
| 지금 주문 | 결제 즉시 매장에서 준비 시작 |
| **예약 주문** 🆕 | 원하는 **날짜·시간 지정** 후 결제. 그 시간에 맞춰 픽업/배달 |

### 예약 주문 흐름
1. 메뉴 담기 → 주문서에서 **수령 방식(픽업/배달)** 선택
2. **예약 주문** 선택 → 날짜·시간 지정 (배달이면 주소 입력)
3. **결제** (§5, 일반 주문과 동일한 결제 수단)
4. 지정 시간에 맞춰 매장이 준비 → 픽업 안내 또는 배달 출발 (푸시 알림)

**규칙**
- 예약 주문도 **결제를 먼저 완료**해야 확정된다. (선결제)
- 예약 가능 시간은 영업시간·정기휴무 범위 안에서만. (초기엔 시간대 목록만 단순 제공)
- 상태: 결제완료 → 준비중 → (픽업대기/배달중) → 완료.

### 배달(직배송) 조건 🆕 (중요)
| 항목 | 규칙 |
| --- | --- |
| 배달 가능 지역 | **명지에코펠리스 건물 내에서만 직배송 가능** |
| 건물 외 | 배달 **불가**, **픽업만** 가능 |
| 픽업 최소 금액 | **없음** (금액 제한 없이 픽업 가능) |
| 직배송 최소 조건 | **샌드위치 2개 이상** 주문 시에만 직배송 가능 |

**동작 규칙**
- 주문서에서 **배달**을 고르면, 배달지가 명지에코펠리스 건물인지 확인(주소 선택/입력 제한).
- 장바구니에 담긴 **샌드위치 수량 합계가 2개 미만**이면 배달 선택을 막고 안내: "직배송은 샌드위치 2개 이상부터 가능해요. 픽업으로 주문해 주세요."
- 이 검증은 화면뿐 아니라 **서버에서도 반드시 재확인**한다(조작 방지).
- 배달비는 초기엔 0원(무료)으로 두고, 필요 시 매장 설정으로 조정.

> 💡 샌드위치 판별을 위해 메뉴에 **카테고리(예: "샌드위치")** 또는 배달조건용 플래그를 둔다(데이터 구조 04 참고).

## 7. 단체 주문 🆕

| 기능 | 설명 |
| --- | --- |
| 단체 문의 | 희망 일시·인원/수량·요청사항·연락처로 문의 접수 |
| 문의 조회 | 마이페이지에서 진행 상태 확인 |

- 즉시 결제가 아니라 **협의형**(사장님이 확인 후 연락).
- 상태: 접수 → 협의중 → 확정 / 취소.

## 9. 푸시 알림 🆕

| 시점 | 알림 내용 |
| --- | --- |
| 주문 상태 변경 | "준비완료됐어요", "배달 출발했어요" 등 |

| 고객의 소리 답변 | 답변 등록 안내 |

**준비물**: 보통 **Firebase Cloud Messaging(FCM)** 사용. 앱에 알림 권한 요청 필요.

## 10. 쿠폰 & 적립 🆕

### 쿠폰
| 기능 | 설명 |
| --- | --- |
| 쿠폰 발급(사장님) | 할인 쿠폰 생성(정액/정률, 유효기간, 사용조건) |
| 쿠폰 사용(손님) | 주문서에서 쿠폰 적용 → 금액 할인 |

### 적립
| 기능 | 설명 |
| --- | --- |
| 적립 지급 | 결제 완료 시 결제금액의 일정 비율 적립 |
| 적립 사용 | 주문서에서 적립금 차감 사용 |

**규칙**
- 쿠폰·적립 할인도 **서버에서 최종 금액 재계산**(조작 방지).
- 쿠폰은 유효기간·중복사용 여부·최소주문금액 조건을 가진다.
- 적립 비율·사용 단위는 매장 설정값(예: 결제금액의 1% 적립).

## 11. 관리자(사장님) 기능 🆕

| 기능 | 설명 |
| --- | --- |
| 관리자 로그인 | 일반 손님과 분리된 **권한(role)** 으로 로그인 |
| 대시보드 | 오늘 주문·매출·신규 예약/문의 요약 |
| 메뉴 관리 | 메뉴 등록·수정·삭제, 품절 토글, 옵션 관리 |
| 주문 관리 | 주문 접수, 상태 변경(→ 손님 알림) |
| 예약/단체/고객의소리 관리 | 각 요청 확인·처리·답변 |
| 쿠폰 관리 | 쿠폰 생성·발급 |

**규칙 (중요)**
- 회원에 **권한 구분**이 필요: 손님(USER) / 사장님(ADMIN).
- 관리자 기능은 반드시 **서버에서 권한 확인** 후 허용(손님이 접근 못 하게).
