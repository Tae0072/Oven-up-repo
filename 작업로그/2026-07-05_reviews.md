## 무엇을 했나 (쉽게)
손님이 먹어본 메뉴에 **별점(1~5)과 후기**를 남길 수 있게 했어요.
메뉴 상세 화면에 평균 별점과 리뷰 목록이 보이고, 아무나 못 쓰게 "구매한 손님만" 작성할 수 있습니다.

## 서버
- `review` 도메인 신규: `ReviewEntity` / `ReviewRepository` / `ReviewService` / `ReviewController`
- `GET /api/menus/{menuId}/reviews` — 리뷰 목록 + 요약(평균·개수) (공개)
- `POST /api/menus/{menuId}/reviews` — 리뷰 작성 (로그인)
  - 구매자만: 결제완료(취소 제외) 주문에 그 메뉴가 있어야 함 → 아니면 400 `NOT_PURCHASED`
  - 메뉴당 1개만 → 중복 400... 아니라 409 `REVIEW_DUPLICATED`
  - 별점 1~5 아니면 400 `INVALID_INPUT`
- 메뉴 응답(`Menu` record)에 `ratingAvg`, `reviewCount` 추가
  - `MenuService`가 `ReviewRepository`로 리뷰를 모아 자바에서 평균·개수 계산(H2/MySQL 무관)
- 테스트 6건(`ReviewControllerTest`): 구매자 작성/비구매자 400/미로그인 401/중복 409/별점범위/메뉴 평점필드

## 앱
- `models/menu_item.dart`: `ratingAvg`, `reviewCount` 추가
- `data/review_api.dart` 신규: 리뷰 목록 조회 + 작성 + 모델(MenuReviews/ReviewView)
- `screens/menu_detail_page.dart`
  - 이름 아래 별점 요약(별 아이콘 + 평균 + 개수)
  - 하단에 리뷰 목록 + `리뷰 쓰기` 버튼
  - 리뷰 작성 바텀시트(별점 탭 + 후기 입력) — 미로그인 시 로그인 유도, 서버가 구매 여부 최종 검증
