# 보안·유지보수 점검 리포트 (2026-07-06)

## 요약
전반적으로 안전하게 구성돼 있음. 발견된 보완점 4가지는 즉시 조치 완료.

## ✅ 이미 잘 되어 있던 것

### 개인정보·보안
- **비밀번호**: BCrypt 해시로 저장 (원문 저장 안 함, 유출돼도 복원 불가)
- **결제**: 카드번호·계좌정보는 우리 서버·앱에 전혀 저장 안 됨 — 이니시스 결제창 안에서만 입력되고,
  서버는 결제 식별자(paymentId)로 PortOne에 "진짜 결제됐는지"만 확인
- **본인인증**: 이름·전화번호만 저장. 주민번호 등 민감정보는 통신사/이니시스가 처리
- **통신 암호화**: 전 구간 HTTPS (Let's Encrypt, 자동갱신)
- **DB 접근**: MySQL이 서버 내부(127.0.0.1)에서만 접속 허용 — 외부에서 DB 직접 접근 불가
- **서버 접속**: SSH 키 전용 (비밀번호 로그인 차단)
- **방화벽**: 22/80/443만 개방 (클라우드 보안목록 + iptables 이중)
- **비밀키 보관**: API 키·DB 비밀번호는 서버 환경파일(권한 600, 소유자만 읽기) / 코드·깃에 없음
- **CI 시크릿 검사**: gitleaks가 PR마다 키 유출 검사
- **관리자 API**: ADMIN 권한 검사 (일반 손님이 호출 시 403)

### 유지보수
- **자동 재시작**: 서버 프로세스 죽으면 10초 후 자동 재시작 (systemd Restart=always)
- **자동 보안업데이트**: Ubuntu unattended-upgrades 활성
- **인증서 자동갱신**: certbot 타이머 동작 중
- **CI/CD**: PR마다 서버·앱 빌드 + 테스트 + 자동머지, 작업로그 문서화
- **디스크**: 14% 사용 (여유 39GB)

## 🔧 이번에 조치한 것 4가지

1. **DB 자동 백업 (없었음 → 신설)**
   - 매일 새벽 4시(KST) 전체 DB를 압축 백업, 14일 보관 (`/opt/ovenup/backups/`)
   - 즉시 1회 실행해 정상 동작 확인
   - 복원: `gunzip < 백업파일 | mysql -uovenup -p ovenup`
2. **CORS 제한 (전체 허용 → 서비스 출처만)** — PR #56
   - 이전: 아무 웹사이트나 우리 API 호출 가능(*)
   - 이후: ven-up.web.app + 로컬 개발만 허용. 검증: 허용 출처 200 / 타 출처 403
3. **웹 보안 헤더 추가** (nginx)
   - HSTS(강제 HTTPS), X-Content-Type-Options, X-Frame-Options(클릭재킹 방지)
4. **앱 서버(8080) 내부 전용 바인딩**
   - 이전엔 모든 인터페이스에 열려 있었음(방화벽이 막고 있긴 했음) → 127.0.0.1 전용으로 이중 방어

## ⚠️ 남은 권고 (실오픈 전 확인)
- **백업의 외부 보관**: 지금은 같은 서버 안에 백업 — VM 자체가 사라지면 같이 소실.
  실오픈 전 오라클 Object Storage(무료 10GB)나 PC로 주기 복사 권장 (요청 시 설정해 드림)
- **메모리 여유 부족(1GB VM)**: 트래픽 늘면 A1 서버(4코어/24GB, 무료)로 이전 필요
- **결제·본인인증 실계약**: 현재 테스트 채널 — 오픈 시 PG 가맹계약 후 채널 교체
- **개인정보처리방침 문서**: 실서비스 오픈 시 법적으로 필요 (웹/앱에 게시). 요청 시 초안 작성 가능
- 관리자 계정(admin@oven.com) 비밀번호를 초기값에서 변경했는지 확인 필요
