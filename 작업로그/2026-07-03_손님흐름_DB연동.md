## 2. 구현 리포트 (DB 연동, PR #10)

### 무엇을 했나
메뉴 데이터를 서버 코드 안 인메모리 목록에서 **DB(JPA)** 로 옮겼다. API 응답 형태(`{success,data}`)와 필드는 그대로라 **앱은 수정 없이** 동작한다.

### 주요 파일 (server/src/main/java/com/ovenup/server/menu/)
- `MenuEntity`, `MenuOptionEntity` — DB 테이블(menu, menu_option) 매핑. `@OneToMany`(menu_id).
- `MenuJpaRepository` — Spring Data JPA 리포지토리.
- `DataInitializer` — 서버 첫 실행 시 메뉴가 비어 있으면 5VEN UP 7종 자동 시드.
- `MenuService` — DB 조회 + 엔티티→DTO(`Menu`) 변환(`@Transactional(readOnly=true)`).

### 결정 사항 (중요)
- **기본 DB는 H2(메모리)** 로 했다. 이유: 입문자가 **아무 설치 없이** 바로 실행/테스트할 수 있고, CI도 이걸로 통과한다. 서버 재시작 시 초기화되지만 시더가 다시 채운다. 콘솔: `http://localhost:8080/h2-console`.
- **MySQL은 프로필로 준비**했다(`application-mysql.properties`). 준비되면 `--spring.profiles.active=mysql` 로 전환. 설정 방법은 `가이드/로컬_MySQL_연결.md`.
- 비밀번호 등은 코드에 넣지 않고 환경변수로 덮어쓰게 했다(`${DB_PASSWORD:}`).

### 검증 결과
- `gradlew build` 성공(ASCII 경로 복사본). 테스트 4건 통과: contextLoads + 메뉴목록(7)/상세(옵션4)/없는메뉴(404).
- 실행 후 실측: `GET /api/menus` → 7종(id 1 = LA갈비 바게트), `GET /api/menus/1` → 옵션 4개(치즈 추가 +1000), `GET /api/menus/999` → 404.
