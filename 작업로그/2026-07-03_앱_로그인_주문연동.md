## 2. 리포트 (PR #14)

### 화면/동작
- **로그인 화면(S1)**: 이메일·비밀번호 로그인, 회원가입(이름·전화) 모드 전환, 에러/로딩 표시. 성공 시 토큰·회원을 `AuthStore`에 저장.
- **메뉴 화면 계정 아이콘**: 로그아웃이면 로그인 화면, 로그인이면 이름·로그아웃 다이얼로그.
- **주문서 [주문하기]**: 미로그인 시 로그인 화면으로 유도 → 로그인 후 `POST /api/orders`로 주문 생성. 성공 시 **주문번호 안내 + 장바구니 비우고 홈으로**, 실패(배달조건 등)면 서버 메시지를 스낵바로 표시.

### 구조 (app/lib/)
- 상태: `state/auth_store.dart`(토큰·회원, ChangeNotifier)
- API: `data/auth_api.dart`(signup/login), `data/order_api.dart`(createOrder), 공용 `data/api_config.dart`·`data/api_exception.dart`
- 모델: `models/auth_user.dart`
- 화면: `screens/login_page.dart`, `menu_list_page`·`order_form_page` 수정

### 보안/설계
- 금액·배달조건·권한은 **서버가 재검증**. 앱은 항목(메뉴/수량/옵션)만 전송하고 금액을 보내지 않는다.
- ⚠️ 로그인 상태는 지금 **앱 메모리 보관**(앱을 완전히 끄면 로그아웃). 이후 `shared_preferences`로 기기에 저장 예정.

### 검증
- `flutter analyze` 통과(No issues found), `flutter test` 통과(4건). 서버 API는 이전 PR(#12/#13)에서 MySQL 실측 완료.
- 전체 연동 확인: 서버(bootRun) + 앱(`flutter run -d chrome`) → 담기 → 주문하기 → 로그인 → 주문번호 발급.
