## 무엇을 했나 (쉽게)
손님이 "지금 말고 이따가/내일 특정 시간에 받고 싶어요"를 앱에서 직접 할 수 있게 만들었어요.
홈의 **예약 주문** 버튼을 누르면 전용 화면이 열리고, 날짜와 시간(30분 단위)을 눈으로 골라
미리 주문할 수 있습니다. 아무 시간이나 되면 안 되니까, 서버가 **영업시간(10~20시)과 미래 시각**인지
한 번 더 확인합니다.

## 서버
- `OrderService`에 예약 시간 검증 추가
  - 주문 요청에 `scheduledAt`이 있으면 검사
  - 과거 시각이면 `400 INVALID_RESERVATION` ("예약 시간은 현재보다 이후여야 해요.")
  - 영업시간(10시~20시) 밖이면 `400 INVALID_RESERVATION` ("예약은 영업시간(10시~20시) 안에서만 가능해요.")
  - 상수 `OPEN_HOUR=10`, `CLOSE_HOUR=20` (20시 시작 슬롯은 불가)
- 테스트 3건 추가(`OrderControllerTest`)
  - 미래·영업시간 내 → 201
  - 과거 → 400 INVALID_RESERVATION
  - 영업시간 밖(새벽 3시) → 400 INVALID_RESERVATION

## 앱
- `screens/reservation_page.dart` 신규 (S9)
  - 장바구니 요약 표시
  - 수령 방식: 포장/배달 (배달은 샌드위치 2개 이상 + 주소 필요)
  - 날짜 선택: 오늘~2주, 가로 칩
  - 시간 선택: 10:00~19:30 30분 단위 칩. 오늘이면 지난 시간은 비활성화
  - 선택 요약 배지 + 요청사항 + 결제 예정금액
  - [예약하고 결제] → 로그인 확인 → 주문 생성(`scheduledAt` 포함) → 결제 화면(S7)
  - 장바구니가 비어 있으면 "메뉴 담으러 가기" 안내
- `screens/home_page.dart`: 홈 '예약 주문' 바로가기를 예약 화면으로 연결(기존 스낵바 안내 대체)
