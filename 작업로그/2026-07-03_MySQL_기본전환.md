## 2. 리포트

### 코드 변경 (PR #11, dev 머지됨)
- `application.properties` 기본 datasource를 **MySQL**로 (환경변수 override, 비밀번호 리터럴 없음, 로컬 평문 접속 옵션 `useSSL=false&allowPublicKeyRetrieval=true`).
- **테스트는 H2 유지**: `src/test/resources/application.properties` 에 H2 → `gradlew build`/GitHub CI가 MySQL 없이도 통과.
- `application-h2.properties` (h2 프로필): MySQL이 없을 때 `--spring.profiles.active=h2` 로 실행.

### 로컬 MySQL 설치 (이 컴퓨터에 실제 구성)
- MySQL **8.4.9**(무설치 zip)를 `D:\dev\mysql-8.4.9-winx64` 에 풀고, 데이터는 `D:\dev\mysql-data`, 설정은 `D:\dev\my.ini`(포트 3306, utf8mb4).
- 초기화(`--initialize-insecure`)로 **root@localhost 비밀번호 없음**(로컬 전용). DB `ovenup`(utf8mb4) 생성.
- **자동 시작**: `D:\dev\ovenup-mysql-start.vbs` 를 시작프로그램(Startup)에 등록 → 로그인 시 창 없이 자동 실행. (관리자 권한이 없어 Windows 서비스 대신 로그인 자동시작 사용)
- 시작/종료 도우미: `ovenup-mysql-start.vbs` / `ovenup-mysql-stop.bat`.

### 검증
- 서버를 기본(MySQL)으로 실행 → JPA가 `menu`/`menu_option` 테이블 생성, 시더가 7종 삽입. `GET /api/menus` → 7종.
- MySQL 직접 조회: `menu`=7행, `menu_option`=28행.
- **영속성**: 서버를 껐다 다시 켜도 여전히 7종(중복 삽입 없음 — 시더가 count>0이면 건너뜀). H2와 달리 데이터가 남는다.

## 3. 지금 어떻게 쓰나 (요약)

- MySQL은 로그인하면 자동 실행됨(포트 3306, DB `ovenup`, root/비번없음).
- 서버 실행: `cd server && .\gradlew.bat bootRun` → 자동으로 MySQL 사용.
- MySQL이 꺼져 있으면 `D:\dev\ovenup-mysql-start.vbs` 더블클릭. 끄려면 `D:\dev\ovenup-mysql-stop.bat`.
- MySQL 없이 임시로 돌리려면: `.\gradlew.bat bootRun --args='--spring.profiles.active=h2'`

자세한 조작·문제해결은 `가이드/로컬_MySQL_연결.md` 참고.
