## 2. 리포트 (PR #12)

### API (05_API §2)
| 메서드/경로 | 설명 | 성공 | 실패 |
| --- | --- | --- | --- |
| POST /api/auth/signup | 회원가입 | 201 `{userId}` | 비번 8자 미만 400 / 이메일 중복 409 |
| POST /api/auth/login | 로그인 | 200 `{accessToken, user{id,name,role}}` | 실패 401 |
| GET /api/users/me | 내 정보(로그인 필요) | 200 `{id,email,name,phone,role,pointBalance}` | 토큰 없음/무효 401 |

### 설계 요점
- 비밀번호는 **BCrypt 해시**로 저장(평문 금지). 로그인 시 해시 비교.
- 로그인 성공 → **JWT**(HS256, 7일). 시크릿은 환경변수 `JWT_SECRET`, 없으면 개발용 랜덤키(재시작 시 기존 토큰 무효).
- **가벼운 인증 방식**: 전체 Spring Security 필터체인을 쓰지 않고, `AuthTokenFilter`(모든 요청에서 `Authorization: Bearer` 파싱 → 요청 속성 `authUserId`)만 둠. 보호가 필요한 컨트롤러(`/api/users/me`)에서 이 값을 확인. → 기존 메뉴 API/CORS/CI에 영향 없음.
- 공통 에러 응답: `ApiException` + `GlobalExceptionHandler` → `{success:false,error:{code,message}}` + 알맞은 HTTP 상태.

### 주요 파일 (server/src/main/java/com/ovenup/server/)
- `user/UserEntity`(users 테이블)·`UserRepository`·`user/UserController`·`user/dto/MyProfile`
- `auth/JwtProvider`·`AuthTokenFilter`·`AuthService`·`AuthController`·`auth/dto/*`
- `common/ApiException`·`GlobalExceptionHandler`

### 검증
- `gradlew build` 성공(ASCII 복사, H2). 새 테스트 `AuthControllerTest` 5건(회원가입/중복/짧은비번/로그인토큰/내정보) 통과 + 기존 테스트 통과.
- MySQL 실측: signup → userId 발급, login → 토큰 발급, me → 내 정보 반환. `users` 테이블에 행 저장 확인(비번은 해시).

### 보안 메모
- `users.email` 유니크. 비밀번호 평문 미저장. JWT 시크릿·DB 비번은 코드에 없음(환경변수).
- ⚠️ 운영 배포 시 `JWT_SECRET`(32자 이상)을 반드시 환경변수로 지정할 것.
