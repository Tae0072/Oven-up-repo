# 가이드 · 카카오·네이버 소셜 로그인 실제 연결 (입문자용)

> 지금은 **mock 모드**라 앱의 "카카오/네이버로 시작하기" 버튼이 가짜로 로그인됩니다.
> 실제로 연결하려면 아래를 따라 하세요. (카카오/네이버 developers 앱 등록이 필요)

---

## 0. 큰 그림

소셜 로그인은 3단계로 나뉩니다.
1. **앱 등록**: 카카오/네이버 개발자 사이트에 우리 앱을 등록하고 키를 받는다.
2. **앱에서 토큰 받기**: Flutter 앱이 카카오/네이버 로그인 창을 띄워 "사용자 액세스 토큰"을 받는다.
3. **서버가 검증**: 앱이 그 토큰을 우리 서버(`POST /api/auth/social/{provider}`)로 보내면, 서버가 카카오/네이버에 물어 확인하고 우리 회원과 연결한다.

지금 코드에는 3번(서버 검증)이 이미 준비돼 있습니다. 1·2번을 하면 됩니다.

---

## 1. 카카오 앱 등록

1. https://developers.kakao.com → 로그인 → **내 애플리케이션** → 애플리케이션 추가.
2. **앱 키**에서 REST API 키 확인.
3. **카카오 로그인** 활성화 → 동의 항목에서 닉네임(선택 이메일) 설정.
4. 플랫폼/리다이렉트 URI 등록(웹이면 http://localhost 등, 앱이면 패키지명).

## 2. 네이버 앱 등록

1. https://developers.naver.com → 오픈 API 이용 신청.
2. **네아로(네이버 아이디로 로그인)** 사용 설정 → Client ID / Client Secret 발급.
3. 콜백 URL 등록.

> ⚠️ 키(REST 키, Client Secret)는 **코드에 직접 쓰지 말고** 환경변수/보안 저장소에 두세요. 커밋 금지.

---

## 3. 앱에서 사용자 토큰 받기 (Flutter)

- 카카오: `kakao_flutter_sdk` 패키지로 로그인 → `OAuthToken.accessToken`.
- 네이버: `flutter_naver_login` 패키지로 로그인 → accessToken.
- (웹만 먼저면, 각 사의 JavaScript SDK로 토큰을 받아도 됩니다.)

그 토큰을 지금 앱의 `AuthApi.socialLogin(provider: 'kakao'|'naver', accessToken: 받은토큰)` 에 넘기면 끝입니다.
→ 지금은 이 자리에 dev mock 토큰(예: `1001:카카오손님`)이 들어가 있으니, 실제 토큰으로 교체하면 됩니다.

---

## 4. 서버를 실제 모드로 실행

```
cd server
.\gradlew.bat bootRun --args='--app.social.mock=false'
```
또는 `application.properties` 의 `app.social.mock` 을 `false` 로 바꿉니다.
- 이러면 서버가 카카오/네이버 사용자 API로 진짜 토큰을 검증합니다(`RealSocialProfileVerifier`).
- 이 검증에는 **사용자 토큰만** 필요하고 별도 시크릿은 필요 없습니다. 시크릿은 2번(토큰 받기) 단계의 앱 쪽에서만 씁니다.

---

## 5. 동작 정리

- 처음 소셜 로그인 → 자동 회원가입(`isNew=true`), `social_account` 에 연결 저장.
- 다음부터 같은 카카오/네이버 계정 → 같은 회원으로 로그인(`isNew=false`).
- 같은 이메일의 자체 회원이 있으면 그 회원에 연결(선택).

---

## 6. 문제 해결

| 증상 | 해결 |
| --- | --- |
| 401 SOCIAL_TOKEN_INVALID | 토큰이 만료/무효. 앱에서 다시 로그인해 새 토큰. |
| 400 UNSUPPORTED_PROVIDER | provider는 kakao 또는 naver만. |
| mock인데 실제로 연결하고 싶음 | `app.social.mock=false` 로 실행. |
