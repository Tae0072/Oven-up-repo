## 2. mock ↔ real 스위치 (핵심)
서버 `server/src/main/resources/application.properties` 의 플래그로 전환합니다. **기본값은 모두 `true`(mock).**

| 연동 | 플래그 | mock(현재) | real(목표) | real 구현 클래스 |
|---|---|---|---|---|
| 소셜로그인 | `app.social.mock` | `MockSocialProfileVerifier` | `RealSocialProfileVerifier` | `social/RealSocialProfileVerifier.java` |
| 결제 | `app.payment.mock` | `MockPaymentVerifier` | `RealPaymentVerifier` | `payment/RealPaymentVerifier.java` |
| 푸시 | `app.push.mock` | `MockPushSender` | `RealPushSender` | `notification/RealPushSender.java` |

전환 방법: 해당 플래그를 `false`로 두면 Real* 빈이 활성화됩니다(`@ConditionalOnProperty`).
**주의:** 테스트/CI는 mock을 그대로 써야 하니, real 값은 코드에 하드코딩하지 말고 **환경변수/운영 프로필**로만 켠다.

## 3. 연동별 할 일

### 3-1. 카카오·네이버 로그인 (`app.social.mock=false`)
- 참고 가이드: `가이드/소셜로그인_연결.md`
- 준비물: 카카오 개발자 앱 키(REST API 키/Redirect URI), 네이버 개발자 앱 Client ID/Secret
- 서버: `RealSocialProfileVerifier` 는 이미 카카오/네이버 사용자 조회 뼈대가 있음 → 실제 토큰 검증·엔드포인트 확정, 키 주입(환경변수)
- 앱: 지금은 dev mock 토큰을 보냄(`AuthApi.socialLogin`). 실제 카카오/네이버 SDK로 access token을 받아 그대로 서버에 전달하도록 교체
- 검증: 실제 계정으로 로그인 → 우리 회원 자동 생성/연결 확인

### 3-2. 결제 (PG) (`app.payment.mock=false` + `PORTONE_API_SECRET`)
- 참고 가이드: `가이드/결제_연동.md`
- 현재 뼈대: PortOne(구 아임포트) 기준. `RealPaymentVerifier(@Value("${PORTONE_API_SECRET:}"))`
- 준비물: PortOne(또는 택한 PG) 가맹점 가입, API Secret, 채널(카카오페이/토스 등) 설정
- 서버: 결제 검증 로직(결제 금액 == 주문 금액 확인) 완성. 결제 흐름은 이미 있음(주문 생성 → 결제창 → `/api/orders/{id}/pay`)
- 앱: 지금은 결제수단만 고르고 mock 결제. 실제 PG 결제창(SDK/웹뷰) 호출 → 성공 시 paymentRef를 서버로 전달하도록 교체
- 검증: 실제(테스트 모드) 결제 → 금액 검증 통과 → 주문 결제완료 + 적립 지급 확인
- ⚠️ 실제 결제는 돈이 오가므로, 반드시 PG의 **테스트 모드**로 먼저 검증. 실제 거래 실행은 사용자가 직접.

### 3-3. 푸시 알림 (FCM) (`app.push.mock=false`)
- 참고 가이드: `가이드/푸시_알림_FCM.md`
- 현재 뼈대: `RealPushSender` 는 로그만 남김(미구현 TODO)
- 준비물: Firebase 프로젝트 + 서비스 계정 키(JSON), 앱에 FCM SDK
- 해야 할 일:
  1) 손님 기기의 FCM 토큰을 서버에 저장할 **기기 토큰 등록 API + 테이블**(신규)
  2) `RealPushSender` 에서 FCM HTTP v1 API로 해당 토큰에 실제 발송
  3) 앱: FCM 토큰 획득 → 등록 API 호출, 포그라운드/백그라운드 알림 처리
- 참고: 인앱 알림함(저장/조회/배지)은 이미 동작. 알림 끄기(`notifyEnabled`)도 서버에서 반영됨 → FCM도 이 설정을 따르게 유지
