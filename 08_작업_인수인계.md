## 3. 작업 규칙 (반드시 지키기)

- **작업 전 항상 `git pull`** 로 최신화 후 시작.
- 모든 작업은 **`feature/*` 브랜치 → `dev`로 PR**. `main`/`dev` 직접 push 금지(보호됨).
- 커밋 메시지: Conventional Commits (`feat(menu): ...`).
- **코드는 Oven-up 레포에만**, 문서는 Oven-up-repo에만.
- 보안·정합성 핵심(리뷰에서도 체크됨):
  - 금액·권한은 **서버에서 재계산·재검증** (화면 값 신뢰 금지)
  - **배달 조건 서버 검증**: 명지에코팰리스 건물 내 + 샌드위치 2개 이상
  - 관리자(ADMIN) 권한 서버 검증, 시크릿(.env/키) 커밋 금지

## 4. 다음 작업 순서 (로드맵)

01번 계획서의 로드맵 기준. **한 번에 하나씩, 가장 작은 것부터.**

1. **프로젝트 뼈대 세팅** — `server/`(Spring Boot) + `app/`(Flutter) 생성, 로컬 실행 확인
2. **손님 화면 (가짜 데이터)** — 홈 → 메뉴 목록 → 메뉴 상세 → 장바구니 → 주문서
3. **DB + 서버 API** — 04 ERD대로 테이블, 05 API대로 메뉴/주문/회원 API
4. **화면 ↔ 서버 연결** — 진짜 메뉴 표시, 주문 저장, 주문내역
5. **손님 편의 기능** — 예약 주문, 단체 주문, 고객의 소리
6. **관리자 화면** — 메뉴·주문·예약 관리
7. **결제·소셜로그인·푸시·쿠폰/적립**
